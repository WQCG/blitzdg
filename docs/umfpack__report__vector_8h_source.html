67 Purpose:
68 
69  Verifies and prints a dense vector.
70 
71 Returns:
72 
73  UMFPACK_OK if Control [UMFPACK_PRL] <= 2 (the input is not checked).
74 
75  Otherwise:
76 
77  UMFPACK_OK if the vector is valid.

81 Arguments:
82 
83  Int n ; Input argument, not modified.
84 
85  X is a real or complex vector of size n. Restriction: n > 0.
86 
87  double X [n] ; Input argument, not modified. For real versions.
88 
89  A real vector of size n. X must not be (double *) NULL.
90 
91  double Xx [n or 2*n] ; Input argument, not modified. For complex versions.
92  double Xz [n or 0] ; Input argument, not modified. For complex versions.
93 
94  A complex vector of size n, in one of two storage formats.
95  Xx must not be (double *) NULL.
96 
97  If Xz is not (double *) NULL, then Xx [i] is the real part of X (i) and
98  Xz [i] is the imaginary part of X (i). Both vectors are of length n.
99  This is the "split" form of the complex vector X.
100 
101  If Xz is (double *) NULL, then Xx holds both real and imaginary parts,
102  where Xx [2*i] is the real part of X (i) and Xx [2*i+1] is the imaginary
103  part of X (i). Xx is of length 2*n doubles. If you have an ANSI C99
104  compiler with the intrinsic double _Complex type, then Xx can be of
105  type double _Complex in the calling routine and typecast to (double *)
106  when passed to umfpack_*_report_vector (this is untested, however).
107  This is the "merged" form of the complex vector X.
108 
109  Note that all complex routines in UMFPACK V4.4 and later use this same
110  strategy for their complex arguments. The split format is useful for
111  MATLAB, which holds its real and imaginary parts in seperate arrays.
112  The packed format is compatible with the intrinsic double _Complex
113  type in ANSI C99, and is also compatible with SuperLU's method of
114  storing complex matrices. In Version 4.3, this routine was the only
115  one that allowed for packed complex arguments.
116 
117  double Control [UMFPACK_CONTROL] ; Input argument, not modified.
118 
119  If a (double *) NULL pointer is passed, then the default control
120  settings are used. Otherwise, the settings are determined from the
121  Control array. See umfpack_*_defaults on how to fill the Control
122  array with the default settings. If Control contains NaN's, the
123  defaults are used. The following Control parameters are used:
124 
125  Control [UMFPACK_PRL]: printing level.
126 
127  2 or less: no output. returns silently without checking anything.
128  3: fully check input, and print a short summary of its status
129  4: as 3, but print first few entries of the input
