188 Purpose:
189 
190  Given the nonzero pattern of a sparse matrix A in column-oriented form, and
191  a sparsity preserving column pre-ordering Qinit, umfpack_*_qsymbolic
192  performs the symbolic factorization of A*Qinit (or A (:,Qinit) in MATLAB
193  notation). This is identical to umfpack_*_symbolic, except that neither
194  COLAMD nor AMD are called and the user input column order Qinit is used
195  instead. Note that in general, the Qinit passed to umfpack_*_qsymbolic
196  can differ from the final Q found in umfpack_*_numeric. The unsymmetric
197  strategy will perform a column etree postordering done in
198  umfpack_*_qsymbolic and sparsity-preserving modifications are made within
199  each frontal matrix during umfpack_*_numeric. The symmetric
200  strategy will preserve Qinit, unless the matrix is structurally singular.
201 
202  See umfpack_*_symbolic for more information. Note that Ax and Ax are
203  optional. The may be NULL.
204 
205  *** WARNING *** A poor choice of Qinit can easily cause umfpack_*_numeric
206  to use a huge amount of memory and do a lot of work. The "default" symbolic
207  analysis method is umfpack_*_symbolic, not this routine. If you use this
208  routine, the performance of UMFPACK is your responsibility; UMFPACK will
209  not try to second-guess a poor choice of Qinit.
210 
211 Returns:
212 
213  The value of Info [UMFPACK_STATUS]; see umfpack_*_symbolic.
214  Also returns UMFPACK_ERROR_invalid_permuation if Qinit is not a valid
215  permutation vector.
216 
217 Arguments:
218 
219  All arguments are the same as umfpack_*_symbolic, except for the following:
220 
221  Int Qinit [n_col] ; Input argument, not modified.
222 
223  The user's fill-reducing initial column pre-ordering. This must be a
224  permutation of 0..n_col-1. If Qinit [k] = j, then column j is the kth
225  column of the matrix A (:,Qinit) to be factorized. If Qinit is an
226  (Int *) NULL pointer, then COLAMD or AMD are called instead.
227 
228  double Control [UMFPACK_CONTROL] ; Input argument, not modified.
229 
230  If Qinit is not NULL, then only two strategies are recognized:
231  the unsymmetric strategy and the symmetric strategy.
232  If Control [UMFPACK_STRATEGY] is UMFPACK_STRATEGY_SYMMETRIC,
233  then the symmetric strategy is used. Otherwise the unsymmetric
234  strategy is used.
