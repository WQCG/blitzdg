blitzdg
an open-source project aiming to implement parallel discontinuous Galerkin (dg) solvers for common partial differential equations systems using blitz++ for array and tensor manipulations and MPI for distributed parallelism.

93 Purpose:
94 
95  Given a sparse matrix A in column-oriented form, and a symbolic analysis
96  computed by umfpack_*_*symbolic, the umfpack_*_numeric routine performs the
97  numerical factorization, PAQ=LU, PRAQ=LU, or P(R\A)Q=LU, where P and Q are
98  permutation matrices (represented as permutation vectors), R is the row
99  scaling, L is unit-lower triangular, and U is upper triangular. This is
100  required before the system Ax=b (or other related linear systems) can be
101  solved. umfpack_*_numeric can be called multiple times for each call to
102  umfpack_*_*symbolic, to factorize a sequence of matrices with identical
103  nonzero pattern. Simply compute the Symbolic object once, with
104  umfpack_*_*symbolic, and reuse it for subsequent matrices. This routine
105  safely detects if the pattern changes, and sets an appropriate error code.
106 
107 Returns:
108 
109  The status code is returned. See Info [UMFPACK_STATUS], below.
110 
111 Arguments:
112 
113  Int Ap [n_col+1] ; Input argument, not modified.
114 
115  This must be identical to the Ap array passed to umfpack_*_*symbolic.
116  The value of n_col is what was passed to umfpack_*_*symbolic (this is
117  held in the Symbolic object).
118 
119  Int Ai [nz] ; Input argument, not modified, of size nz = Ap [n_col].
120 
121  This must be identical to the Ai array passed to umfpack_*_*symbolic.
122 
123  double Ax [nz] ; Input argument, not modified, of size nz = Ap [n_col].
124  Size 2*nz for packed complex case.
125 
126  The numerical values of the sparse matrix A. The nonzero pattern (row
127  indices) for column j is stored in Ai [(Ap [j]) ... (Ap [j+1]-1)], and
128  the corresponding numerical values are stored in
129  Ax [(Ap [j]) ... (Ap [j+1]-1)].
130 
131  double Az [nz] ; Input argument, not modified, for complex versions.
132 
133  For the complex versions, this holds the imaginary part of A. The
134  imaginary part of column j is held in Az [(Ap [j]) ... (Ap [j+1]-1)].
135 
136  If Az is NULL, then both real
137  and imaginary parts are contained in Ax[0..2*nz-1], with Ax[2*k]
138  and Ax[2*k+1] being the real and imaginary part of the kth entry.
139 
140  void *Symbolic ; Input argument, not modified.
141 
142  The Symbolic object, which holds the symbolic factorization computed by
143  umfpack_*_*symbolic. The Symbolic object is not modified by
144  umfpack_*_numeric.
145 
146  void **Numeric ; Output argument.
147 
148  **Numeric is the address of a (void *) pointer variable in the user's
149  calling routine (see Syntax, above). On input, the contents of this
150  variable are not defined. On output, this variable holds a (void *)
151  pointer to the Numeric object (if successful), or (void *) NULL if
152  a failure occurred.
153 
154  double Control [UMFPACK_CONTROL] ; Input argument, not modified.
155 
156  If a (double *) NULL pointer is passed, then the default control
157  settings are used. Otherwise, the settings are determined from the
158  Control array. See umfpack_*_defaults on how to fill the Control
159  array with the default settings. If Control contains NaN's, the
160  defaults are used. The following Control parameters are used:
161 
162  Control [UMFPACK_PIVOT_TOLERANCE]: relative pivot tolerance for
163  threshold partial pivoting with row interchanges. In any given
164  column, an entry is numerically acceptable if its absolute value is
165  greater than or equal to Control [UMFPACK_PIVOT_TOLERANCE] times
166  the largest absolute value in the column. A value of 1.0 gives true
167  partial pivoting. If less than or equal to zero, then any nonzero
168  entry is numerically acceptable as a pivot. Default: 0.1.
169 
170  Smaller values tend to lead to sparser LU factors, but the solution
171  to the linear system can become inaccurate. Larger values can lead
172  to a more accurate solution (but not always), and usually an
173  increase in the total work.
174 
175  For complex matrices, a cheap approximate of the absolute value
176  is used for the threshold partial pivoting test (|a_real| + |a_imag|
177  instead of the more expensive-to-compute exact absolute value
178  sqrt (a_real^2 + a_imag^2)).
179 
180  Control [UMFPACK_SYM_PIVOT_TOLERANCE]:
181  If diagonal pivoting is attempted (the symmetric
182  strategy is used) then this parameter is used to control when the
183  diagonal entry is selected in a given pivot column. The absolute
184  value of the entry must be >= Control [UMFPACK_SYM_PIVOT_TOLERANCE]
185  times the largest absolute value in the column. A value of zero
186  will ensure that no off-diagonal pivoting is performed, except that
187  zero diagonal entries are not selected if there are any off-diagonal
188  nonzero entries.
189 
190  If an off-diagonal pivot is selected, an attempt is made to restore
191  symmetry later on. Suppose A (i,j) is selected, where i != j.
192  If column i has not yet been selected as a pivot column, then
193  the entry A (j,i) is redefined as a "diagonal" entry, except that
194  the tighter tolerance (Control [UMFPACK_PIVOT_TOLERANCE]) is
195  applied. This strategy has an effect similar to 2-by-2 pivoting
196  for symmetric indefinite matrices. If a 2-by-2 block pivot with
197  nonzero structure
198 
199  i j
200  i: 0 x
201  j: x 0
202 
203  is selected in a symmetric indefinite factorization method, the
204  2-by-2 block is inverted and a rank-2 update is applied. In
205  UMFPACK, this 2-by-2 block would be reordered as
206 
207  j i
208  i: x 0
209  j: 0 x
210 
211  In both cases, the symmetry of the Schur complement is preserved.
212 
213  Control [UMFPACK_SCALE]: Note that the user's input matrix is
214  never modified, only an internal copy is scaled.
215 
216  There are three valid settings for this parameter. If any other
217  value is provided, the default is used.
218 
219  UMFPACK_SCALE_NONE: no scaling is performed.
220 
221  UMFPACK_SCALE_SUM: each row of the input matrix A is divided by
222  the sum of the absolute values of the entries in that row.
223  The scaled matrix has an infinity norm of 1.
224 
225  UMFPACK_SCALE_MAX: each row of the input matrix A is divided by
226  the maximum the absolute values of the entries in that row.
227  In the scaled matrix the largest entry in each row has
228  a magnitude exactly equal to 1.
229 
230  Note that for complex matrices, a cheap approximate absolute value
231  is used, |a_real| + |a_imag|, instead of the exact absolute value
232  sqrt ((a_real)^2 + (a_imag)^2).
233 
234  Scaling is very important for the "symmetric" strategy when
235  diagonal pivoting is attempted. It also improves the performance
236  of the "unsymmetric" strategy.
237 
238  Default: UMFPACK_SCALE_SUM.
239 
240  Control [UMFPACK_ALLOC_INIT]:
241 
242  When umfpack_*_numeric starts, it allocates memory for the Numeric
243  object. Part of this is of fixed size (approximately n double's +
244  12*n integers). The remainder is of variable size, which grows to
245  hold the LU factors and the frontal matrices created during
246  factorization. A estimate of the upper bound is computed by
247  umfpack_*_*symbolic, and returned by umfpack_*_*symbolic in
248  Info [UMFPACK_VARIABLE_PEAK_ESTIMATE] (in Units).
249 
250  If Control [UMFPACK_ALLOC_INIT] is >= 0, umfpack_*_numeric initially
251  allocates space for the variable-sized part equal to this estimate
252  times Control [UMFPACK_ALLOC_INIT]. Typically, for matrices for
253  which the "unsymmetric" strategy applies, umfpack_*_numeric needs
254  only about half the estimated memory space, so a setting of 0.5 or
255  0.6 often provides enough memory for umfpack_*_numeric to factorize
256  the matrix with no subsequent increases in the size of this block.
257 
258  If the matrix is ordered via AMD, then this non-negative parameter
259  is ignored. The initial allocation ratio computed automatically,
260  as 1.2 * (nz + Info [UMFPACK_SYMMETRIC_LUNZ]) /
261  (Info [UMFPACK_LNZ_ESTIMATE] + Info [UMFPACK_UNZ_ESTIMATE] -
262  min (n_row, n_col)).
263 
264  If Control [UMFPACK_ALLOC_INIT] is negative, then umfpack_*_numeric
265  allocates a space with initial size (in Units) equal to
266  (-Control [UMFPACK_ALLOC_INIT]).
267 
268  Regardless of the value of this parameter, a space equal to or
269  greater than the the bare minimum amount of memory needed to start
270  the factorization is always initially allocated. The bare initial
271  memory required is returned by umfpack_*_*symbolic in
272  Info [UMFPACK_VARIABLE_INIT_ESTIMATE] (an exact value, not an
273  estimate).
274 
275  If the variable-size part of the Numeric object is found to be too
276  small sometime after numerical factorization has started, the memory
277  is increased in size by a factor of 1.2. If this fails, the
278  request is reduced by a factor of 0.95 until it succeeds, or until
279  it determines that no increase in size is possible. Garbage
280  collection then occurs.
281 
282  The strategy of attempting to "malloc" a working space, and
283  re-trying with a smaller space, may not work when UMFPACK is used
284  as a mexFunction MATLAB, since mxMalloc aborts the mexFunction if it
285  fails. This issue does not affect the use of UMFPACK as a part of
286  the built-in x=A\b in MATLAB 6.5 and later.
287 
288  If you are using the umfpack mexFunction, decrease the magnitude of
289  Control [UMFPACK_ALLOC_INIT] if you run out of memory in MATLAB.
290 
291  Default initial allocation size: 0.7. Thus, with the default
292  control settings and the "unsymmetric" strategy, the upper-bound is
293  reached after two reallocations (0.7 * 1.2 * 1.2 = 1.008).
294 
295  Changing this parameter has little effect on fill-in or operation
296  count. It has a small impact on run-time (the extra time required
297  to do the garbage collection and memory reallocation).
298 
299  Control [UMFPACK_FRONT_ALLOC_INIT]:
300 
301  When UMFPACK starts the factorization of each "chain" of frontal
302  matrices, it allocates a working array to hold the frontal matrices
303  as they are factorized. The symbolic factorization computes the
304  size of the largest possible frontal matrix that could occur during
305  the factorization of each chain.
306 
307  If Control [UMFPACK_FRONT_ALLOC_INIT] is >= 0, the following
308  strategy is used. If the AMD ordering was used, this non-negative
309  parameter is ignored. A front of size (d+2)*(d+2) is allocated,
310  where d = Info [UMFPACK_SYMMETRIC_DMAX]. Otherwise, a front of
311  size Control [UMFPACK_FRONT_ALLOC_INIT] times the largest front
312  possible for this chain is allocated.
313 
314  If Control [UMFPACK_FRONT_ALLOC_INIT] is negative, then a front of
315  size (-Control [UMFPACK_FRONT_ALLOC_INIT]) is allocated (where the
316  size is in terms of the number of numerical entries). This is done
317  regardless of the ordering method or ordering strategy used.
318 
319  Default: 0.5.
320 
321  Control [UMFPACK_DROPTOL]:
322 
323  Entries in L and U with absolute value less than or equal to the
324  drop tolerance are removed from the data structures (unless leaving
325  them there reduces memory usage by reducing the space required
326  for the nonzero pattern of L and U).
327 
328  Default: 0.0.
