99 Purpose:
100 
101  Given nonzero pattern of a sparse matrix A in column-oriented form,
102  umfpack_*_symbolic performs a column pre-ordering to reduce fill-in
103  (using COLAMD, AMD or METIS) and a symbolic factorization. This is required
104  before the matrix can be numerically factorized with umfpack_*_numeric.
105  If you wish to bypass the COLAMD/AMD/METIS pre-ordering and provide your own
106  ordering, use umfpack_*_qsymbolic instead. If you wish to pass in a
107  pointer to a user-provided ordering function, use umfpack_*_fsymbolic.
108 
109  Since umfpack_*_symbolic and umfpack_*_qsymbolic are very similar, options
110  for both routines are discussed below.
111 
112  For the following discussion, let S be the submatrix of A obtained after
113  eliminating all pivots of zero Markowitz cost. S has dimension
114  (n_row-n1-nempty_row) -by- (n_col-n1-nempty_col), where
115  n1 = Info [UMFPACK_COL_SINGLETONS] + Info [UMFPACK_ROW_SINGLETONS],
116  nempty_row = Info [UMFPACK_NEMPTY_ROW] and
117  nempty_col = Info [UMFPACK_NEMPTY_COL].
118 
119 Returns:
120 
121  The status code is returned. See Info [UMFPACK_STATUS], below.
122 
123 Arguments:
124 
125  Int n_row ; Input argument, not modified.
126  Int n_col ; Input argument, not modified.
127 
128  A is an n_row-by-n_col matrix. Restriction: n_row > 0 and n_col > 0.
129 
130  Int Ap [n_col+1] ; Input argument, not modified.
131 
132  Ap is an integer array of size n_col+1. On input, it holds the
133  "pointers" for the column form of the sparse matrix A. Column j of
134  the matrix A is held in Ai [(Ap [j]) ... (Ap [j+1]-1)]. The first
135  entry, Ap [0], must be zero, and Ap [j] <= Ap [j+1] must hold for all
136  j in the range 0 to n_col-1. The value nz = Ap [n_col] is thus the
137  total number of entries in the pattern of the matrix A. nz must be
138  greater than or equal to zero.
139 
140  Int Ai [nz] ; Input argument, not modified, of size nz = Ap [n_col].
141 
142  The nonzero pattern (row indices) for column j is stored in
143  Ai [(Ap [j]) ... (Ap [j+1]-1)]. The row indices in a given column j
144  must be in ascending order, and no duplicate row indices may be present.
145  Row indices must be in the range 0 to n_row-1 (the matrix is 0-based).
146  See umfpack_*_triplet_to_col for how to sort the columns of a matrix
147  and sum up the duplicate entries. See umfpack_*_report_matrix for how
148  to print the matrix A.
149 
150  double Ax [nz] ; Optional input argument, not modified. May be NULL.
151  Size 2*nz for packed complex case.
152 
153  The numerical values of the sparse matrix A. The nonzero pattern (row
154  indices) for column j is stored in Ai [(Ap [j]) ... (Ap [j+1]-1)], and
155  the corresponding numerical values are stored in
156  Ax [(Ap [j]) ... (Ap [j+1]-1)]. Used only for gathering statistics
157  about how many nonzeros are placed on the diagonal by the fill-reducing
158  ordering.
159 
160  double Az [nz] ; Optional input argument, not modified, for complex
161  versions. May be NULL.
162 
163  For the complex versions, this holds the imaginary part of A. The
164  imaginary part of column j is held in Az [(Ap [j]) ... (Ap [j+1]-1)].
165 
166  If Az is NULL, then both real
167  and imaginary parts are contained in Ax[0..2*nz-1], with Ax[2*k]
168  and Ax[2*k+1] being the real and imaginary part of the kth entry.
169 
170  Used for statistics only. See the description of Ax, above.
171 
172  void **Symbolic ; Output argument.
173 
174  **Symbolic is the address of a (void *) pointer variable in the user's
175  calling routine (see Syntax, above). On input, the contents of this
176  variable are not defined. On output, this variable holds a (void *)
177  pointer to the Symbolic object (if successful), or (void *) NULL if
178  a failure occurred.
179 
180  double Control [UMFPACK_CONTROL] ; Input argument, not modified.
181 
182  If a (double *) NULL pointer is passed, then the default control
183  settings are used (the defaults are suitable for all matrices,
184  ranging from those with highly unsymmetric nonzero pattern, to
185  symmetric matrices). Otherwise, the settings are determined from the
186  Control array. See umfpack_*_defaults on how to fill the Control
187  array with the default settings. If Control contains NaN's, the
188  defaults are used. The following Control parameters are used:
189 
190  Control [UMFPACK_STRATEGY]: This is the most important control
191  parameter. It determines what kind of ordering and pivoting
192  strategy that UMFPACK should use. There are 4 options:
193 
194  UMFPACK_STRATEGY_AUTO: This is the default. The input matrix is
195  analyzed to determine how symmetric the nonzero pattern is, and
196  how many entries there are on the diagonal. It then selects one
197  of the following strategies. Refer to the User Guide for a
198  description of how the strategy is automatically selected.
199 
200  UMFPACK_STRATEGY_UNSYMMETRIC: Use the unsymmetric strategy. COLAMD
201  is used to order the columns of A, followed by a postorder of
202  the column elimination tree. No attempt is made to perform
203  diagonal pivoting. The column ordering is refined during
204  factorization.
205 
206  In the numerical factorization, the
207  Control [UMFPACK_SYM_PIVOT_TOLERANCE] parameter is ignored. A
208  pivot is selected if its magnitude is >=
209  Control [UMFPACK_PIVOT_TOLERANCE] (default 0.1) times the
210  largest entry in its column.
211 
212  UMFPACK_STRATEGY_SYMMETRIC: Use the symmetric strategy
213  In this method, the approximate minimum degree
214  ordering (AMD) is applied to A+A', followed by a postorder of
215  the elimination tree of A+A'. UMFPACK attempts to perform
216  diagonal pivoting during numerical factorization. No refinement
217  of the column pre-ordering is performed during factorization.
218 
219  In the numerical factorization, a nonzero entry on the diagonal
220  is selected as the pivot if its magnitude is >= Control
221  [UMFPACK_SYM_PIVOT_TOLERANCE] (default 0.001) times the largest
222  entry in its column. If this is not acceptable, then an
223  off-diagonal pivot is selected with magnitude >= Control
224  [UMFPACK_PIVOT_TOLERANCE] (default 0.1) times the largest entry
225  in its column.
226 
227  Control [UMFPACK_ORDERING]: The ordering method to use:
228  UMFPACK_ORDERING_CHOLMOD try AMD/COLAMD, then METIS if needed
229  UMFPACK_ORDERING_AMD just AMD or COLAMD
230  UMFPACK_ORDERING_GIVEN just Qinit (umfpack_*_qsymbolic only)
231  UMFPACK_ORDERING_NONE no fill-reducing ordering
232  UMFPACK_ORDERING_METIS just METIS(A+A') or METIS(A'A)
233  UMFPACK_ORDERING_BEST try AMD/COLAMD, METIS, and NESDIS
234  UMFPACK_ORDERING_USER just user function (*_fsymbolic only)
235 
236  Control [UMFPACK_SINGLETONS]: If false (0), then singletons are
237  not removed prior to factorization. Default: true (1).
238 
239  Control [UMFPACK_DENSE_COL]:
240  If COLAMD is used, columns with more than
241  max (16, Control [UMFPACK_DENSE_COL] * 16 * sqrt (n_row)) entries
242  are placed placed last in the column pre-ordering. Default: 0.2.
243 
244  Control [UMFPACK_DENSE_ROW]:
245  Rows with more than max (16, Control [UMFPACK_DENSE_ROW] * 16 *
246  sqrt (n_col)) entries are treated differently in the COLAMD
247  pre-ordering, and in the internal data structures during the
248  subsequent numeric factorization. Default: 0.2.
249 
250  Control [UMFPACK_AMD_DENSE]: rows/columns in A+A' with more than
251  max (16, Control [UMFPACK_AMD_DENSE] * sqrt (n)) entries
252  (where n = n_row = n_col) are ignored in the AMD pre-ordering.
253  Default: 10.
254 
255  Control [UMFPACK_BLOCK_SIZE]: the block size to use for Level-3 BLAS
256  in the subsequent numerical factorization (umfpack_*_numeric).
257  A value less than 1 is treated as 1. Default: 32. Modifying this
258  parameter affects when updates are applied to the working frontal
259  matrix, and can indirectly affect fill-in and operation count.
260  Assuming the block size is large enough (8 or so), this parameter
261  has a modest effect on performance.
262 
263  Control [UMFPACK_FIXQ]: If > 0, then the pre-ordering Q is not modified
264  during numeric factorization. If < 0, then Q may be modified. If
265  zero, then this is controlled automatically (the unsymmetric
266  strategy modifies Q, the others do not). Default: 0.
267 
268  Control [UMFPACK_AGGRESSIVE]: If nonzero, aggressive absorption is used
269  in COLAMD and AMD. Default: 1.
