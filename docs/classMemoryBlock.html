Allocate a memory block.

If we're allocating a large array it may be desireable for performance reasons to force the array to start on a cache line boundary. We do this by allocating a little more memory than necessary, then shifting the pointer to the next cache line boundary. Note that this only works if the SIMD byte width of P_type fits evenly into the cache line size, otherwise we may have to allocate a lot more memory to be sure to get to a common multiple of both the cache line size and the size of the object. This can easily be the case for multicomponent containers, and in this case, nothing is done.x
